## 📋 Dataset

| Attribute | Value |
|-----------|-------|
| Records | 180,519 orders |
| Customers | 20,652 unique |
| Products | 118 |
| Categories | 50 |
| Markets | 5 (Europe, LATAM, Pacific Asia, USCA, Africa) |
| Total Revenue | $36,784,734 |
| Total Profit | $3,966,903 |
| Overall Margin | 10.78% |

### Key Findings
- **Discount-to-Profit Ratio: 94%** — discounts of $3.73M consume 94% of net profit
- **18.71%** of orders are loss-making (33,784 transactions)
- **Top 10% customers** account for **49.1%** of total profit
- Pearson r = −0.0027 (p = 0.253) at order level; monotonic 25.2% margin decline across discount bands
- First Class shipping: **95.32% late rate** — anomaly requiring audit
- 2 pp discount reduction = **$674,419** profit recovery (+17.0%)

## 🧪 Data Transformation Pipeline

**`data_transformation.py`** performs 8 operations:
1. Latin-1 CSV ingestion (180,519 records, 40 fields)
2. Complete column renaming to snake_case
3. String normalisation & categorical standardisation
4. Removal of 4 spatial columns (street, zipcode, lat, lon)
5. Non-positive sales record validation (0 removed)
6. **43 engineered features** — financial, shipping, discount, customer tiers, product tiers, market/category aggregates
7. Customer, product, market, category aggregation with order-level left-join
8. Output: `APL_Logistics_Transformed.csv` (180,519 rows × 79 columns)
